# 数据库设计

## 概述

knowledge-dao 使用 PostgreSQL 15+ 作为主数据库，通过 `pgvector` 扩展支持向量存储与相似度检索。

- 向量维度: **1024** (bge-m3 模型)
- 向量索引: **HNSW** (余弦距离)
- 用户隔离: 所有表按 `user_id` 隔离查询

## 表结构总览

| 表名 | 用途 | 核心字段 |
|------|------|---------|
| `knowledge_base` | 知识库主表，含向量 | id, title, content, embedding |
| `chat_messages` | 对话历史 | id, session_key, role, content |
| `user_profile` | 用户配置 KV | user_id, key, value (JSONB) |
| `shared_knowledge` | 跨用户共享知识 | 同 knowledge_base |

## knowledge_base

主知识库表，按用户隔离，支持向量语义检索。

```sql
CREATE TABLE IF NOT EXISTS knowledge_base (
    id              BIGSERIAL PRIMARY KEY,
    title           VARCHAR(500) NOT NULL,
    content         TEXT NOT NULL,
    content_type    VARCHAR(100),          -- article, book, note, doc ...
    tags            JSONB,                 -- ["java", "programming"]
    embedding       VECTOR(1024),          -- 向量，HNSW 索引
    source          VARCHAR(500),          -- 来源标识
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    user_id         BIGINT NOT NULL,
    is_shared       BOOLEAN NOT NULL DEFAULT FALSE
);
```

### 索引

```sql
-- 用户隔离索引
CREATE INDEX idx_knowledge_base_user ON knowledge_base(user_id);

-- 时间排序索引
CREATE INDEX idx_knowledge_base_user_created ON knowledge_base(user_id, created_at DESC);

-- HNSW 向量索引 (余弦距离)
CREATE INDEX idx_knowledge_base_embedding
  ON knowledge_base USING hnsw (embedding vector_cosine_ops);
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 自增主键 |
| `title` | VARCHAR(500) | 知识标题 |
| `content` | TEXT | 知识正文内容 |
| `content_type` | VARCHAR(100) | 内容类型，如 article/book/note |
| `tags` | JSONB | 标签数组，JSON 数组格式 |
| `embedding` | VECTOR(1024) | bge-m3 生成的 1024 维向量 |
| `source` | VARCHAR(500) | 来源，可用于溯源 |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |
| `user_id` | BIGINT | 所属用户 ID |
| `is_shared` | BOOLEAN | 是否共享给其他用户 |

## chat_messages

对话历史表，按会话（session）分组。

```sql
CREATE TABLE IF NOT EXISTS chat_messages (
    id          BIGSERIAL PRIMARY KEY,
    session_key VARCHAR(255) NOT NULL,   -- 会话唯一标识
    role        VARCHAR(50) NOT NULL,     -- user / assistant / system
    content     TEXT NOT NULL,
    created_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    user_id     BIGINT NOT NULL
);
```

### 索引

```sql
CREATE INDEX idx_chat_messages_session
  ON chat_messages(session_key, user_id);

CREATE INDEX idx_chat_messages_user_created
  ON chat_messages(user_id, created_at);
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 自增主键 |
| `session_key` | VARCHAR(255) | 会话 ID，如 "session-001" |
| `role` | VARCHAR(50) | 角色: user/assistant/system |
| `content` | TEXT | 消息内容 |
| `created_at` | TIMESTAMP | 发送时间 |
| `user_id` | BIGINT | 所属用户 |

## user_profile

用户配置键值存储表，value 为 JSONB，支持任意结构。

```sql
CREATE TABLE IF NOT EXISTS user_profile (
    key         VARCHAR(255) NOT NULL,
    value       JSONB NOT NULL,
    updated_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    user_id     BIGINT NOT NULL,
    PRIMARY KEY (key, user_id)
);
```

### 索引

```sql
CREATE INDEX idx_user_profile_user ON user_profile(user_id);
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | VARCHAR(255) | 配置键 |
| `value` | JSONB | 配置值，任意 JSON 对象 |
| `updated_at` | TIMESTAMP | 更新时间 |
| `user_id` | BIGINT | 用户 ID (复合主键之一) |

### 示例数据

```sql
-- 字符串值
INSERT INTO user_profile VALUES ('name', '"张三"', NOW(), 1);

-- 对象值
INSERT INTO user_profile VALUES ('preferences', '{"theme":"dark","language":"zh"}', NOW(), 1);

-- 数组值
INSERT INTO user_profile VALUES ('tags', '["技术","AI"]', NOW(), 1);
```

## shared_knowledge

跨用户共享的知识条目，用于公共知识库场景。

```sql
CREATE TABLE IF NOT EXISTS shared_knowledge (
    id              BIGSERIAL PRIMARY KEY,
    owner_user_id   BIGINT NOT NULL,
    title           VARCHAR(500) NOT NULL,
    content         TEXT NOT NULL,
    content_type    VARCHAR(100),
    tags            JSONB,
    embedding       VECTOR(1024),
    source          VARCHAR(500),
    created_at      TIMESTAMP NOT NULL DEFAULT NOW()
);
```

### 索引

```sql
CREATE INDEX idx_shared_knowledge_owner
  ON shared_knowledge(owner_user_id);

CREATE INDEX idx_shared_knowledge_embedding
  ON shared_knowledge USING hnsw (embedding vector_cosine_ops);
```

## pgvector 扩展

### 安装

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 向量格式

pgvector 使用 `[v1,v2,...,v1024]` 格式存储向量：

```sql
-- 手动插入向量示例
INSERT INTO knowledge_base (title, content, embedding, user_id)
VALUES ('Test', 'Content', '[0.1,0.2,0.3,...]'::vector, 1);
```

### 距离计算

| 操作符 | 说明 |
|--------|------|
| `<=>` | 余弦距离 (cosine distance) |
| `<->` | L2 欧氏距离 |
| `<#>` | 负内积 |

本项目使用 `<=>`（余弦距离），配合 `vector_cosine_ops` 操作符。

### HNSW 索引参数调优

```sql
-- 可选：调整 HNSW 参数（构建速度 vs 检索精度）
CREATE INDEX idx_kb_embedding
  ON knowledge_base USING hnsw (embedding vector_cosine_ops)
  WITH (m = 16, ef_construction = 128);
```

- `m`: 每个节点的最大连接数，默认 16
- `ef_construction`: 构建时的动态列表大小，默认 128
