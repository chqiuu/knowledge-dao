# 架构设计

## 系统概览

knowledge-dao 是一个轻量级 RAG（Retrieval-Augmented Generation）知识库系统，核心目标是**将知识文档向量化后存入 PostgreSQL，通过语义相似度检索为 AI Agent 提供知识增强能力**。

系统分为四层架构：

```
┌─────────────────────────────────────────────────────┐
│                  App (应用层)                        │
│  CLI 测试入口 / 未来 HTTP API (Javalin)             │
└──────────────────────┬────────────────────────────┘
                       │
┌──────────────────────▼────────────────────────────┐
│               Service Layer (服务层)               │
│  ┌──────────────────┐    ┌──────────────────────┐  │
│  │ EmbeddingService │    │    RagService        │  │
│  │  向量化服务       │    │  RAG 检索编排        │  │
│  │  Ollama API 调用  │    │  buildContext()      │  │
│  └────────┬─────────┘    └──────────┬───────────┘  │
└───────────┼───────────────────────────┼─────────────┘
            │                           │
┌───────────▼───────────────────────────▼────────────┐
│             Store Layer (存储抽象层)                │
│         VectorStore 接口 (可扩展实现)                │
│              │                                     │
│    ┌────────┴────────┐                            │
│    │  PgvectorStore  │                            │
│    │  向量 → pgvector │                            │
│    └─────────────────┘                            │
└──────────────────────┬────────────────────────────┘
                       │
┌──────────────────────▼────────────────────────────┐
│              DAO Layer (数据访问层)                 │
│  KnowledgeDao  │  ChatMessageDao  │ UserProfileDao │
└──────────────────────┬────────────────────────────┘
                       │
┌──────────────────────▼────────────────────────────┐
│           PostgreSQL + pgvector                    │
│  • 关系型数据存储 (chat_messages, user_profile)     │
│  • 向量相似度检索 (knowledge_base.embedding)        │
└────────────────────────────────────────────────────┘
```

## 组件关系

### EmbeddingService

负责调用 Ollama REST API，将文本转为 1024 维浮点向量：

```
文本 → Ollama /api/embeddings → float[1024]
```

- 单条: `embed(String text)` → `float[]`
- 批量: `embedBatch(List<String> texts)` → `List<float[]>`
- 内置 `cosineSimilarity(float[], float[])` 计算余弦相似度
- `toPgVectorString(float[])` 将向量转为 `[v1,v2,...,v1024]` 格式

### RagService

RAG 编排层，封装检索逻辑：

| 方法 | 说明 |
|------|------|
| `retrieve(query, userId, topK)` | 获取 topK 条最相关知识 |
| `retrieveWithThreshold(...)` | 增加相似度阈值过滤 |
| `buildContext(query, userId, topK)` | 构造成 LLM prompt 可用的上下文字符串 |
| `addKnowledge(...)` | 新增知识条目（自动生成向量） |

### KnowledgeDao

知识库主表 CRUD + 向量检索：

```
insert()    → 自动调用 VectorStore.embed() 生成向量，存入 pgvector
search()    → ORDER BY embedding <=> query_vector (余弦距离)
searchWithEntries() → search() 的完整 Entry 版本
findAll()   → 流式读取，支持大规模数据集
streamAll() → callback 模式，避免全量加载
update()    → 重新生成向量
delete()    → 按 id + userId 删除（防串读）
```

### ChatMessageDao

对话历史管理，每个 Session 独立存储：

```
insert()          → 新增消息
findBySession()   → 按 sessionKey 读取会话历史
findRecent()      → 最近 N 小时内消息
countBySession()  → 统计会话消息数
deleteOlderThan() → 清理 N 天前历史
findDistinctSessions() → 获取用户所有会话
```

### UserProfileDao

用户配置 KV 存储，使用 PostgreSQL JSONB：

```
set(key, value)  → UPSERT，value 可为任意 JSON 序列化对象
get(key)         → 读取并反序列化
exists(key)      → 检查 key 是否存在
delete(key)      → 删除
```

## 数据流

### 知识入库流程

```
用户输入 (title + content)
        ↓
EmbeddingService.embed(title + " " + content)
        ↓
float[1024] 向量
        ↓
KnowledgeDao.insert()
        ↓
SQL: INSERT ... embedding = ?::vector
     → pgvector 存储向量 [v1,v2,...,v1024]
```

### 语义检索流程

```
用户查询 "什么是 Java"
        ↓
EmbeddingService.embed("什么是 Java")
        ↓
float[1024] 查询向量
        ↓
KnowledgeDao.search()
        ↓
SQL: SELECT ... ORDER BY embedding <=> '查询向量' LIMIT topK
     → pgvector 使用 HNSW 索引加速余弦距离计算
        ↓
List<KnowledgeEntry> 按相似度排序返回
```

### RAG Context 构建流程

```
RagService.buildContext("什么是 Java", userId, 3)
        ↓
EmbeddingService.embed("什么是 Java")  [query 向量]
        ↓
KnowledgeDao.searchWithEntries()      [topK=3]
        ↓
List<KnowledgeEntry>
        ↓
格式化输出:
  ## Relevant Knowledge (Top 3 results):
  ### [1] Java Programming Guide
  Java is a high-level, class-based...
  Tags: java, programming, tutorial
  ...
```

## 向量索引策略

knowledge_base 表使用 **HNSW 索引**：

```sql
CREATE INDEX idx_knowledge_base_embedding
  ON knowledge_base USING hnsw (embedding vector_cosine_ops);
```

- `hnsw`: Hierarchical Navigable Small World，图索引，适合高维向量近似最近邻
- `vector_cosine_ops`: 使用余弦距离计算
- 优势：检索速度快（对 1024 维向量效果良好）
- 劣势：构建索引慢，存储开销较大

## 配置加载

```
config.yaml (或环境变量)
       ↓
Config.load() [线程安全单例]
       ↓
EmbeddingService(Config)
       ↓
PgvectorStore(EmbeddingService)
       ↓
KnowledgeDao(Connection, PgvectorStore)
```

密码优先从 `DB_PASSWORD` 环境变量读取，fallback 到 config.yaml。

## 扩展点

- **VectorStore 接口**: 可实现 MilvusStore、ChromaStore 等替代 pgvector
- **Config**: 支持从 classpath 或当前目录加载 YAML
- **Connection**: 由 HikariCP 管理连接池，支持配置池大小
