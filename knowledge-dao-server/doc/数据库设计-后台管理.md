## 1. 扩展概述

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| 可审计 | 所有写操作可追溯到人、时间、IP |
| 可观测 | API 性能、错误率可量化监控 |
| 可管理 | 支持管理员账户体系，与业务用户分离 |
| 可扩展 | 新增表结构不影响现有业务 |

### 1.2 新增对象一览

| 对象 | 类型 | 用途 |
|------|------|------|
| `operation_logs` | 表 | 知识库写操作审计日志 |
| `api_metrics` | 表 | API 请求性能指标 |
| `admin_users` | 表 | 后台管理员账户 |
| `admin_operation_log` | 表 | 管理员自身操作审计 |
| `cleanup_old_logs()` | 函数 | 清理过期操作日志 |
| `cleanup_old_api_metrics()` | 函数 | 清理过期 API 指标 |

## 2. 现有表扩展

### 2.1 knowledge_base 新增字段

```sql
ALTER TABLE knowledge_base ADD COLUMN status VARCHAR(20) NOT NULL DEFAULT 'active';
```

| 值 | 含义 |
|----|------|
| `active` | 正常（默认） |
| `inactive` | 禁用（管理员手动禁用） |
| `deleted` | 已删除（软删除） |

> 软删除策略：物理数据保留，仅改 status，不影响向量索引。

### 2.2 chat_messages 新增字段

```sql
ALTER TABLE chat_messages ADD COLUMN status VARCHAR(20) NOT NULL DEFAULT 'active';
```

| 值 | 含义 |
|----|------|
| `active` | 正常（默认） |
| `deleted` | 已删除（软删除） |

### 2.3 shared_knowledge 新增字段

```sql
ALTER TABLE shared_knowledge ADD COLUMN status VARCHAR(20) NOT NULL DEFAULT 'active';
```

## 3. 新增表

### 3.1 operation_logs（操作日志）

**用途：** 记录所有针对知识库的写操作，供安全审计和数据追溯。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 主键 |
| `action` | VARCHAR(50) | 操作类型：create/update/delete/share/unshare/restore/export |
| `target_type` | VARCHAR(50) | 目标类型：knowledge_entry/chat_message/user_profile/shared_knowledge |
| `target_id` | BIGINT | 目标记录 ID |
| `target_title` | VARCHAR(500) | 目标标题（方便排查） |
| `operator_id` | BIGINT | 操作用户 ID |
| `ip_address` | VARCHAR(45) | 客户端 IP（支持 IPv6） |
| `user_agent` | VARCHAR(500) | User-Agent |
| `request_method` | VARCHAR(10) | HTTP 方法 |
| `request_path` | VARCHAR(500) | 请求路径 |
| `request_params` | JSONB | 请求参数（脱敏） |
| `response_status` | SMALLINT | HTTP 响应码 |
| `error_message` | TEXT | 错误信息 |
| `created_at` | TIMESTAMP | 操作时间 |

**action 枚举说明：**

| action | 说明 |
|--------|------|
| `create` | 新建知识/消息 |
| `update` | 更新知识/消息 |
| `delete` | 删除（软删除） |
| `share` | 共享知识 |
| `unshare` | 取消共享 |
| `restore` | 恢复已删除记录 |
| `export` | 导出数据 |

### 3.2 api_metrics（API 指标）

**用途：** 记录每个 API 请求的性能数据，用于 SLA 监控和慢查询分析。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 主键 |
| `endpoint` | VARCHAR(255) | API 路径 |
| `method` | VARCHAR(10) | HTTP 方法 |
| `status_code` | SMALLINT | HTTP 响应码 |
| `latency_ms` | INTEGER | 响应耗时（毫秒） |
| `user_id` | BIGINT | 请求用户（未登录为 NULL） |
| `ip_address` | VARCHAR(45) | 客户端 IP |
| `error_type` | VARCHAR(50) | 错误类型分类 |
| `request_size` | INTEGER | 请求体大小（字节） |
| `response_size` | INTEGER | 响应体大小（字节） |
| `created_at` | TIMESTAMP | 请求时间 |

**error_type 枚举：**

| 值 | 含义 |
|----|------|
| `timeout` | 请求超时 |
| `validation` | 参数校验失败 |
| `auth` | 认证/授权失败 |
| `server_error` | 服务器内部错误 |
| `null` | 无错误 |

### 3.3 admin_users（管理员账户）

**用途：** 独立的管理员认证体系，与业务用户（knowledge_base.user_id）分离。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 主键 |
| `username` | VARCHAR(100) | 用户名（唯一） |
| `password_hash` | VARCHAR(255) | bcrypt 密码哈希 |
| `role` | VARCHAR(50) | 角色：super_admin/admin/viewer |
| `nickname` | VARCHAR(100) | 昵称 |
| `email` | VARCHAR(255) | 邮箱（唯一） |
| `is_active` | BOOLEAN | 是否启用 |
| `last_login_at` | TIMESTAMP | 最后登录时间 |
| `last_login_ip` | VARCHAR(45) | 最后登录 IP |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

**role 权限说明：**

| 角色 | 权限 |
|------|------|
| `super_admin` | 超级管理员，可管理所有管理员账户 |
| `admin` | 管理员，可操作知识库和查看指标 |
| `viewer` | 查看者，仅可查看数据和日志 |

### 3.4 admin_operation_log（管理员审计）

**用途：** 记录管理员自身的管理操作（如登录、配置变更）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL | 主键 |
| `admin_id` | BIGINT | 管理员 ID |
| `action` | VARCHAR(100) | 操作类型 |
| `target_type` | VARCHAR(50) | 目标类型（可选） |
| `target_id` | BIGINT | 目标 ID（可选） |
| `detail` | JSONB | 操作详情（灵活扩展） |
| `ip_address` | VARCHAR(45) | 客户端 IP |
| `user_agent` | VARCHAR(500) | User-Agent |
| `created_at` | TIMESTAMP | 操作时间 |

## 4. 索引设计

### 4.1 索引总览

| 表 | 索引名 | 字段 | 用途 |
|----|--------|------|------|
| `knowledge_base` | `idx_kb_user_status_created` | (user_id, status, created_at DESC) | 用户+状态过滤+时间排序 |
| `knowledge_base` | `idx_kb_status_created` | (status, created_at DESC) | 全库状态统计 |
| `chat_messages` | `idx_cm_session_created` | (session_key, user_id, created_at DESC) | 会话历史查询 |
| `chat_messages` | `idx_cm_user_created_status` | (user_id, created_at DESC) | 用户消息列表 |
| `operation_logs` | `idx_ol_action_created` | (action, created_at DESC) | 操作类型统计/审计 |
| `operation_logs` | `idx_ol_target` | (target_type, target_id, created_at DESC) | 单条记录操作历史 |
| `operation_logs` | `idx_ol_operator` | (operator_id, created_at DESC) | 管理员操作记录 |
| `operation_logs` | `idx_ol_created` | (created_at DESC) | 全表时间扫描 |
| `api_metrics` | `idx_am_endpoint_created` | (endpoint, created_at DESC) | 接口性能统计 |
| `api_metrics` | `idx_am_user_created` | (user_id, created_at DESC) | 用户调用统计 |
| `api_metrics` | `idx_am_status_created` | (status_code, created_at DESC) WHERE status >= 400 | 错误率监控 |
| `api_metrics` | `idx_am_latency` | (latency_ms DESC, created_at DESC) WHERE latency > 1000 | 慢查询定位 |
| `api_metrics` | `idx_am_created` | (created_at DESC) | 全表时间扫描 |
| `admin_users` | `idx_au_username` | (username) | 登录查询 |
| `admin_users` | `idx_au_role_active` | (role, is_active) | 角色过滤 |
| `admin_operation_log` | `idx_aol_admin_created` | (admin_id, created_at DESC) | 管理员操作历史 |
| `admin_operation_log` | `idx_aol_action_created` | (action, created_at DESC) | 操作类型统计 |
| `user_profile` | `idx_up_key_updated` | (key, updated_at DESC) | key 枚举查询 |
| `user_profile` | `idx_up_updated` | (updated_at DESC) | 时间范围查询 |
| `shared_knowledge` | `idx_sk_status_created` | (status, created_at DESC) WHERE status != 'deleted' | 软删除过滤 |

### 4.2 常用查询索引对应

| 查询场景 | 索引 |
|---------|------|
| 检索历史：user_id + created_at | `idx_cm_user_created_status` |
| 操作日志：action + created_at | `idx_ol_action_created` |
| API 指标：endpoint + created_at | `idx_am_endpoint_created` |
| 知识库：user_id + status + created_at | `idx_kb_user_status_created` |

### 4.3 软删除索引（Partial Index）

部分索引只索引未删除的记录，减少索引体积：

```sql
-- 知识库软删除过滤
CREATE INDEX idx_knowledge_base_user_status_created
    ON knowledge_base(user_id, status, created_at DESC)
    WHERE status != 'deleted';

-- 聊天记录软删除过滤
CREATE INDEX idx_chat_messages_session_created
    ON chat_messages(session_key, user_id, created_at DESC)
    WHERE status != 'deleted';

-- API 错误请求
CREATE INDEX idx_api_metrics_status_created
    ON api_metrics(status_code, created_at DESC)
    WHERE status_code >= 400;

-- API 慢查询（>1s）
CREATE INDEX idx_api_metrics_latency
    ON api_metrics(latency_ms DESC, created_at DESC)
    WHERE latency_ms > 1000;
```

## 5. 数据保留策略

### 5.1 保留周期

| 数据类型 | 默认保留 | 说明 |
|---------|---------|------|
| `operation_logs` | 90 天 | 通过 `cleanup_old_logs()` 函数清理 |
| `api_metrics` | 30 天 | 通过 `cleanup_old_api_metrics()` 函数清理 |
| `admin_operation_log` | 永久 | 管理员操作需长期保留 |
| `chat_messages` | 用户可配置 | 默认永久，deleteOldThan() 支持手动清理 |
| `knowledge_base` | 永久 | 物理删除需管理员操作 |

### 5.2 清理函数

```sql
-- 清理 90 天前的操作日志
SELECT cleanup_old_logs(90);

-- 清理 30 天前的 API 指标
SELECT cleanup_old_api_metrics(30);
```

建议通过 cron job 定期执行：
```sql
-- 每周日凌晨 3 点执行清理
0 3 * * 0 SELECT cleanup_old_logs(90);
0 3 * * 0 SELECT cleanup_old_api_metrics(30);
```
