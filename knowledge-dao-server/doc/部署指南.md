# 部署指南

## 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | 推荐 OpenJDK 17 LTS |
| Maven | 3.8+ | 构建工具 |
| PostgreSQL | 15+ | 必须启用 pgvector 扩展 |
| Ollama | latest | 向量化模型服务 |

### 硬件建议

- **内存**: 最低 4GB，推荐 8GB+
- **CPU**: 支持 AVX 指令集（大多数现代 CPU 均支持）
- **磁盘**: 取决于知识库规模，每千条知识条目约 ~4MB（1024维 float 向量）

## 1. 安装 PostgreSQL + pgvector

### macOS

```bash
brew install postgresql@15
brew install pgvector   # 需要 PostgreSQL 已安装

# 启动
brew services start postgresql@15
```

### Linux (Ubuntu/Debian)

```bash
# 安装 PostgreSQL
sudo apt update
sudo apt install postgresql postgresql-contrib

# 安装 pgvector
sudo apt install postgresql-15-pgvector
# 或从源码编译（针对旧版本 PostgreSQL）
```

### 初始化数据库

```bash
# 以 postgres 用户创建数据库
sudo -u postgres createdb knowledge

# 创建用户（可选）
sudo -u postgres createuser dou
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE knowledge TO dou;"

# 启用 pgvector
psql -d knowledge -c "CREATE EXTENSION IF NOT EXISTS vector;"

# 执行建表脚本
psql -d knowledge -f schema.sql
```

### 验证 pgvector

```sql
psql -d knowledge -c "SELECT extname, extversion FROM pg_extension WHERE extname = 'vector';"
```

## 2. 安装 Ollama

### macOS / Linux

```bash
# 安装
curl -fsSL https://ollama.com/install.sh | sh

# 启动服务
ollama serve

# 拉取 bge-m3 模型（约 0.67GB）
ollama pull bge-m3
```

### 验证

```bash
curl http://localhost:11434/api/embeddings -d '{
  "model": "bge-m3",
  "prompt": "Hello world"
}'
```

正常返回 `{ "embedding": [...] }`。

## 3. 项目配置

编辑 `src/main/resources/config.yaml`：

```yaml
db:
  host: localhost
  port: 5432
  database: knowledge
  username: dou
  password: chqiuu2026        # 生产环境建议通过 DB_PASSWORD 环境变量

ollama:
  base-url: http://localhost:11434
  embedding-model: bge-m3

app:
  embedding-dimension: 1024
```

### 生产环境配置建议

**数据库密码**:
```bash
export DB_PASSWORD="your_secure_password"
java -jar knowledge-dao-1.0.0.jar
```

**连接池调优** (修改 HikariCP 配置):
```java
hikariConfig.setMaximumPoolSize(20);   // 默认 10
hikariConfig.setMinimumIdle(5);        // 默认 2
hikariConfig.setConnectionTimeout(30000);
hikariConfig.setIdleTimeout(300000);
```

## 6. Docker 部署（可选）

### Dockerfile

```dockerfile
FROM eclipse-temurin:17-jre-alpine

WORKDIR /app
COPY target/knowledge-dao-1.0.0.jar app.jar
COPY schema.sql /app/schema.sql
COPY src/main/resources/config.yaml /app/config.yaml

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### Docker Compose

```yaml
version: '3.8'
services:
  postgres:
    image: ankane/pgvector:latest
    environment:
      POSTGRES_DB: knowledge
      POSTGRES_USER: dou
      POSTGRES_PASSWORD: chqiuu2026
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./schema.sql:/docker-entrypoint-initdb.d/schema.sql

  ollama:
    image: ollama/ollama:latest
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]

  app:
    build: .
    depends_on:
      - postgres
      - ollama
    environment:
      DB_PASSWORD: chqiuu2026

volumes:
  pgdata:
  ollama_data:
```

## 7. 生产环境注意事项

### 数据库安全

- **网络**: PostgreSQL 仅监听本地或内网，不暴露公网
- **密码**: 使用 `DB_PASSWORD` 环境变量，不写在配置文件中
- **权限**: 遵循最小权限原则，应用用户不授予 DDL 权限

### 向量索引维护

- HNSW 索引在大数据量（>10万条）时构建较慢，提前规划
- 新增数据不影响现有索引，无需重建

### Ollama 优化

- GPU 加速：ollama 自动使用 NVIDIA GPU，无 GPU 则用 CPU
- 批量处理：`EmbeddingService.embedBatch()` 远优于循环调用单条接口
- 模型缓存：`ollama pull` 后模型常驻内存，无需每次启动拉取

### 日志

项目使用 SLF4J + Simple，配置 `simplelogger.properties`:

```properties
org.slf4j.simpleLogger.defaultLogLevel=info
org.slf4j.simpleLogger.log.com.knowledge=debug
```

## 8. 常见问题

### Ollama 连接失败

```bash
# 确认服务运行
curl http://localhost:11434/api/tags

# 如未运行
ollama serve
```

### pgvector 向量维度不匹配

检查 `config.yaml` 中 `embedding-dimension` 是否为 1024，与 Ollama 模型输出一致。

### 内存溢出 (OOM)

调整 HikariCP 池大小或 JVM heap：

```bash
java -Xmx2g -jar knowledge-dao-1.0.0.jar
```

### 数据库连接被拒绝

确认 PostgreSQL 服务运行中：

```bash
# macOS
brew services list | grep postgresql

# Linux
sudo systemctl status postgresql
```
